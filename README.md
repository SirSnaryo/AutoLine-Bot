<pre> # Discord AutoLine Bot A simple and configurable Discord bot made with JavaScript and discord.js v14. This bot automatically sends a PNG file when a message matches your setup. You can make it work in specific channels, trigger it with custom words, limit it to specific users, add a cooldown, delete the user's message, and control the bot status from the config file. ## Features - Automatically sends a PNG file - Works in selected channels or all channels - Can trigger on specific words - Optional user whitelist - Optional reply to the message - Optional delete user message - Configurable cooldown - Custom bot status - Optional credits status - Loads and caches the PNG file on startup for faster sending ## Requirements - Node.js 18 or newer - A Discord bot token - Message Content Intent enabled in the Discord Developer Portal ## Installation Install the dependencies: npm install ## Project Files Your project should look like this: index.js config.yml package.json line.png If you want to use a different PNG file name, change it in config.yml. ## Configuration Edit config.yml before starting the bot. Example: TOKEN: "-" # Put your Bot Token Here GUILD_ID: "-" # Put your Guild ID Here, keep "-" to allow all guilds BOT_STATUS_ENABLED: false # Enable or Disable Your Custom Bot Status Here CREDITS_STATUS: false # Show Developer Credits Status Here BOT_STATUS_TYPE: "PLAYING" # PLAYING, WATCHING, LISTENING, COMPETING BOT_STATUS_TEXT: "-" # Your Custom Status Text Here LINE_FILE_NAME: "line.png" # PNG file name, must be in the same folder as index.js AUTO_LINE_CHANNELS_ENABLED: false # If true, any message in the channels below sends the line AUTO_LINE_CHANNEL_IDS: - "000000000000000000" - "000000000000000000" AUTO_LINE_WORDS_ENABLED: false # If true, message must contain one of the words below AUTO_LINE_WORDS: - "line" - "tt" AUTO_LINE_WHITELIST_ENABLED: false # If true, only users below can trigger the line AUTO_LINE_WHITELISTED_USERS: - "000000000000000000" AUTO_LINE_REPLY_TO_MESSAGE: false # If true, bot replies to the message instead of normal send DELETE_USER_MESSAGE: false # If true, bot deletes the user message when sending the line AUTO_LINE_COOLDOWN: 0 # Cooldown in seconds per user, 0 = disabled ## Important Notes - Keep all IDs in quotes - Put the PNG file in the same folder as index.js - If AUTO_LINE_CHANNELS_ENABLED is true, any message in those channels can trigger the line - If AUTO_LINE_WORDS_ENABLED is true, only messages containing one of the configured words can trigger the line - If both channel and word options are enabled, the bot will send when either condition matches - If DELETE_USER_MESSAGE is enabled, the bot needs Manage Messages permission - If AUTO_LINE_REPLY_TO_MESSAGE is enabled and the user message is deleted, the bot will send normally instead of replying ## Bot Permissions Make sure the bot has these permissions: - View Channels - Send Messages - Attach Files - Read Message History - Manage Messages if DELETE_USER_MESSAGE is enabled ## Enable Message Content Intent This bot reads message content for trigger words, so you need to enable Message Content Intent. Go to: 1. Discord Developer Portal 2. Your Application 3. Bot 4. Enable Message Content Intent ## Start the Bot Run: node index.js Or: npm start ## Console Output The bot prints clear logs in the console. Examples: Logged in as "BotName" Loaded "line.png" and cached it successfully Sent line in "general" - Reason: "AutoLine Channel" Sent line in "chat" - Reason: "AutoLine Word "line"" Failed to send line. User is not whitelisted Failed to send line. The file "line.png" was not found in the bot folder ## Troubleshooting ### The bot logs in but does not send the line Check these: - The bot has permission to send files - The PNG file exists - Your channel IDs are correct - Your user IDs are correct - Message Content Intent is enabled - The bot is in the correct server - IDs in config.yml are wrapped in quotes ### Word trigger works, but channel trigger does not Make sure: - AUTO_LINE_CHANNELS_ENABLED is set to true - The channel ID is correct - The channel ID is wrapped in quotes ### The bot cannot delete messages Make sure the bot has: - Manage Messages permission ## Dependencies - discord.js 14.26.4 - js-yaml 4.1.0 ## License This project is for
